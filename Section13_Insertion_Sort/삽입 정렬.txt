# 삽입 정렬

 - 기본적으로 버블 정렬, 선택 정렬과 비슷하지만 장점이 있음

 - 배열의 과반을 점차적으로 만들어 정렬을 구축하며, 과반은 항상 정렬되어 있음

 - 한 요소에서 크거나 작은 요소를 찾는게 아닌, 각 요소를 취하여 정렬되어 있는 절반에 해당하는 위치에 배치함

# 삽입 정렬 의사코드

 - 배열에서 두 번째 인덱스를 선택하여 시작

 - 두 번째 값을 취해서 앞에 있는 값과 비교하고 필요하면 스왑

 - 그런 다음 옆의 요소로 이동하여 올바른 위치에 있는지 확인

 - 반복하면서 정렬


# 삽입 정렬의 시간복잡도

 - 기본적으로 O(n^2) => 순서가 완전하게 역방향인 경우 최악

 - 이미 어느정도 정렬된 케이스에서 다른 기본 알고리즘 보다 좋음

 - 데이터가 들어오는 대로 작동하는 온라인 알고리즘 데이터가 있을 경우도 유리
