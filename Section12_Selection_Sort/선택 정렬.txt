# 선택 정렬

 - 큰 값을 배열의 끝에 위치시키는 버블 정렬과 달리 작은 값을 한 번에 하나씩 위치에 배열

 - 최소값 요소와 선택된 값을 교환

# 선택 정렬 의사 코드

 - 최소값을 저장할 변수를 만들고 배열의 맨 처음 값을 최소값으로 지정

 - 최소값과 배열의 다음 항목을 어느 것이 더 작은지 비교

 - 만약 더 작은 값을 발견했다면 최소값을 갱신하고 갱신된 최소값으로 배열의 끝까지 탐색

 - 첫 시작 최소값과 갱신된 최소값이 다르면 교환

 - 정렬된 항목을 제하고 배열 끝까지 정렬

# 선택 정렬 시간 복잡도

 - 기본적으로 O(n^2)

 - 일반적인 상황에서는 버블 정렬보다 못하지만 어떤 이유로 메모리에 쓰는 것을 고려하거나
   
   실제 스왑을 수행하는 것을 고려하는 경우(흔하지는 않음) 사용할 수 있음 - 게다가 쉽다!
